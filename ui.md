# OFDM Host UI 设计与功能基线

本文档是当前 Flutter UI 的唯一基线说明。后续凡是 UI 相关改动（布局、视觉、交互、文案、组件、行为），都直接更新本文件。

## 1. 文档用途

- 固化当前 UI 设计与交互行为，避免口头约定丢失。
- 作为 UI 回归和评审依据：改动前后对照本文件。
- 与 README 的迁移目标联动，记录已完成与待优化项。

## 2. 当前 UI 技术实现

- 前端框架：Flutter Desktop（Windows 优先）
- 主题体系：Material 3
- 字体：Google Fonts 的 Space Grotesk
- 窗口管理：window_manager（无边框窗口、拖动区域、窗口关闭控制）
- 主页面实现文件：flutter_ui/lib/src/app.dart
- 入口文件：flutter_ui/lib/main.dart

## 3. 当前视觉设计（已实现）

### 3.1 视觉方向

- 明亮暖色系基调，避免传统控制台的纯黑背景压迫感。
- 页面背景使用三色渐变：暖橙 -> 清绿 -> 米白。
- 控件区块采用圆角卡片 + 半透明白底，突出信息层级。
- 窗口最外边界使用大圆角（半径 20），形成一体化桌面应用轮廓。

### 3.2 色彩与信息层级

- 全局主色种子：#E36F2D
- 页面背景：#F8D3A7 / #BDE6D8 / #F2EFE9 渐变
- 状态/指标深色卡片：#102A43 系列
- 波形主色：offset 蓝色、delay 橙色
- 高亮命中：日志关键词使用黄底高对比高亮

### 3.3 动效与反馈

- 顶部状态卡使用 AnimatedContainer 过渡。
- 状态文本使用 AnimatedSwitcher 切换，减少突变感。
- 按钮状态、分析中状态、导出成功路径都有即时反馈。

## 4. 当前布局结构（已实现）

### 4.1 响应式断点

- 断点规则：宽度 < 1100 进入紧凑布局；否则进入双栏布局。
- 紧凑布局：顶部状态 + 垂直滚动（控制区 -> 波形区 -> 日志区）。
- 宽屏布局：左侧固定控制区，右侧依次显示状态栏、波形区、日志区。

### 4.2 页面分区

- 顶部状态栏（Header）
  - 标题：OFDM Host · Flutter 控制台
  - 服务状态文本
  - 三个指标 Chip：丢包、offset、delay
- 左侧控制面板（Control Panel）
  - 服务启动参数、串口控制、记录控制、离线分析与导出
- 中部可视化面板（Visualization）
  - 实时 offset / delay 双波形
  - 统计值（mean/min/max/samples）
  - 波形窗口、Y 缩放、自动/固定量程控制
- 底部事件日志面板（Log Panel）
  - 实时事件列表
  - 关键词过滤
  - 命中高亮

### 4.3 桌面窗口顶栏（新增）

- 隐藏 Windows 原生白色标题栏（最小化/最大化/关闭原生按钮不再显示）。
- 不使用独立的窗口条，保持页面视觉一体化。
- 整个窗口外层采用圆角裁剪，超出圆角范围的内容会被裁剪。
- 顶部状态区支持拖动整个窗口。
- 顶部状态区保留自定义关闭按钮替代原生关闭按钮。

## 5. 当前功能清单（已实现）

### 5.1 连接与服务控制

- 设置 Python 命令和 core_service.py 路径
- 启动/停止 Python Core Service
- Windows 端使用无边框窗口，实现 Flutter 一体化顶部 UI
- 顶部状态区整体可拖动窗口并提供自定义关闭按钮
- 最外层窗口边界为大圆角（radius=20）并裁剪溢出内容
- 启动前自动执行路径自检（core_service.py / simulate_input.txt）
- 路径缺失时自动回退到候选路径（含 python_core 发布目录）
- 实时串口列表刷新
- 真实串口/模拟串口模式切换
- 模拟文件路径配置
- 真实串口模式下串口下拉框使用大圆角样式

### 5.2 采集与记录控制

- 打开/关闭串口
- 开始/停止记录
- 支持“等待触发词后开始记录”
- 记录根目录与备注可配置
- 显示 parsed/raw 输出文件路径

### 5.3 离线分析

- 输入历史文件路径后触发分析
- trim 比例滑条（0~20%）
- 结果卡片展示：offset/delay 的 mean/min/max、used/raw、converge_time
- 分析结果导出：TXT / JSON
- 导出目录可配置（默认 analysis_exports）
- 导出命名模板可配置（支持 {type} / {ts} / {date}）
- 重名策略可配置（自动重命名 / 覆盖 / 跳过）

### 5.4 实时可视化

- offset 与 delay 双波形同屏展示
- 时间窗切换：120 / 240 / 420 / 800 / 1200 点
- Y 轴缩放：0.5x ~ 4.0x
- 量程策略：自动量程 / 固定量程
- 固定量程支持 offset 与 delay 独立滑条设置（±0.001 ~ ±0.2）
- 支持清空波形

### 5.5 日志与调试

- 显示服务事件与错误输出
- 日志总条数限制（防止无限增长）
- 支持关键词过滤
- 支持过滤命中高亮
- 已增加高亮缓存过滤与限量渲染，降低高流量卡顿
- 支持清空日志
